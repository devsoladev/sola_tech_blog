## Type in TypeScript

### Basic

- boolean <br />
  참 또는 거짓을 나타내는 타입

  ```typescript
  let isDone: boolean = true;
  let isLoggedIn: boolean = false;
  ```

- number <br />
  숫자를 나타내는 타입

  ```typescript
  let num: number = 1;
  ```

- string <br />
  문자열을 나타내는 타입

```typescript
let str: string = 'hello';
```

- array <br />
  배열을 나타내는 타입

```typescript
let arr: number[] = [1, 2, 3];
```

- tuple <br />
  요소의 타입과 개수가 고정된 배열을 나타내는 타입

```typescript
let tuple: [string, number];
tuple = ['hello', 1];
```

- enum <br />
  유한한 경우의 수를 가지는 값의 집합을 나타내는 타입

```typescript
enum Direction {
	East = 2,
	West,
	South,
	North,
}

let here: Direction = Direction.South;
let here: Direction = Direction[2]; // East
let here: Direction = Direction[5]; // North
```

- any <br />
  모든 값을 허용하는 타입

```typescript
let anyType: any = 10;
anyType = 'ten';
anyType = false;
```

- void <br />
  아무런 값도 반환하지 않는 함수의 반환 타입

```typescript
function nothing(): void {}
```

- null <br />
  null값을 가지는 타입

```typescript
let nullVal: null = null;
```

- undefined <br />
  undefined값을 가지는 타입

```typescript
let undefinedVal: undefined = undefined;
```

- never <br />
  절대 발생할 수 없는 타입

```typescript
// never를 반환하는 함수는 절대 마지막에 도달하지 못한다.
function error(msg: string): never {
	throw new Error(msg);
}

function infiniteLoop(): never {
	while (true) {}
}
```

- object <br />
  객체 자료 구조를 의미하는 타입

```typescript
const user: {
	name: string;
	age: number;
} = {
	name: 'sola',
	age: 20,
};
```

### Type Assertion

타입 단언은 개발자가 해당 타입에 확신을 가질 때 사용하는 타입 지정 방식이다. 타 언어의 '타입 변환'과 유사한 개념으로, 컴파일할 때 특별히 타입 검사를 하거나 데이터를 재구성하지 않는다.

```typescript
let some: any = 'string type';

// angle-bracket
let strLength: number = (<string>some).length;

// as-
let srtLength: number = (some as string).length;
```

### Type Aliases

타입 별칭은 특정 타입이나 인터페이스를 참조할 수 있는 타입 변수이다. 이미 존재하는 타입에 별칭을 붙여 복잡한 타입을 간단하게 사용할 수 있다.

```typescript
type Person = {
	name: string;
	age: number;
};

let sola: Person;
```
